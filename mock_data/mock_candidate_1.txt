--- INTERVIEW Q&A ---

Q1: What is fullstack?
A:  Fullstack means working on both the frontend and backend of an application.
    The frontend is what users see in the browser and the backend handles server
    logic, APIs, and databases. A fullstack developer can build an entire feature
    end to end without depending on a separate team for either side. In my current
    role at Wipro I work across both layers using React on the frontend and
    Node.js with PostgreSQL on the backend.

Q2: What is the difference between frontend and backend?
A:  Frontend is everything the user directly sees and interacts with — the
    layout, buttons, forms, and animations rendered in the browser using HTML,
    CSS, and JavaScript. Backend is the server side that processes requests,
    applies business logic, reads and writes to databases, and sends responses
    back to the frontend. Neither works in isolation; the frontend calls the
    backend through APIs and the backend returns data the frontend then displays.

Q3: What is React?
A:  React is a JavaScript library created by Facebook for building user
    interfaces. It uses a component based model where you break the UI into
    small, reusable pieces. React maintains a virtual DOM and only re-renders
    the parts of the page that actually changed, which makes it very efficient.
    I have used React for building dashboards and data-heavy interfaces and find
    its ecosystem — hooks, context, and libraries like React Query — very
    productive for real-world projects.
