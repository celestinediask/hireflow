--- INTERVIEW Q&A ---

Q1: What is fullstack?
A:  Fullstack means working on both the frontend and the backend of a web
    application. The frontend is what the user sees and interacts with in
    the browser, and the backend handles the data, business logic, and
    server-side operations. A fullstack developer can take a feature from
    the database all the way to the user interface. In my current role I
    do this regularly — I work on React components for the UI and write
    Node.js APIs that serve data to those components.

Q2: What is the difference between frontend and backend?
A:  Frontend is the client-side code that runs in the browser. It includes
    HTML for structure, CSS for styling, and JavaScript for making things
    interactive. Frameworks like React or Vue help manage complex UIs.
    Backend is server-side and handles things the user never directly sees —
    authentication, database queries, business logic, and sending data to
    the frontend through APIs. They communicate over HTTP usually using
    JSON. I have worked on both sides so I understand how they fit together,
    though I am more confident on the backend side from my earlier Java work.

Q3: What is React?
A:  React is a JavaScript library from Meta for building user interfaces.
    It works with a component-based model where you break the UI into small
    reusable pieces. Each component manages its own state and React re-renders
    only the parts that change, which makes it efficient. I use hooks like
    useState and useEffect regularly. One thing I had to get used to was the
    unidirectional data flow — props go down and events go up. I have not
    used Redux much; in our project we use React Context for shared state
    which has been sufficient so far.
